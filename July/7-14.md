### 커밋 컨벤션
- `커밋 카테고리`
    - `feat` → 새로운 기능 추가
    - `fix` → 현재 릴리즈 되지 않은 상태에서 수정
    - `hotfix` → 현재 릴리즈 된 상태인 코드에서 수정(배포상태에서 바로 코드 수정해서 다시 릴리즈 하기 위함)
    - `chore` → 코드 기능상에 변화가 없는 잡(?)탕 수정 (ex. .gitignore 수정)
    - `style` → 코드 js에서는 린트 스타일 이나 코드 컨벤션 수정
    - `docs` → README 와 같은 md 파일 수정
    - `test` → 테스트 코드 작성
