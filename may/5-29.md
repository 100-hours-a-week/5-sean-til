### 인덱스

→ 특정 컬럼들에 대해 인덱스를 생성함으로써 레코드를 빠르게 찾는다. 풀스캔을 하면 안됌.

create index 사용, 항상쓴다.

update, delete 쿼리는 실행되는 시점에 row를 검색한다(select가 실행됨

SELECT 쿼리 순서

1. FROM 절
2. JOIN 테이블 결합
3. WHERE 조건 필터링
4. GROUP BY 그룹화
5. HAVING 그룹화된 결과 필터링
6. SELECT 출력할 컬럼 선택
7. DISTINCT 중복제거
8. ORDER BY 정렬
9. LIMIT 반환할 레코드 수 제한
